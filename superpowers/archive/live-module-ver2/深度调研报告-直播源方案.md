# 直播源问题深度调研报告

> 调研日期: 2026-03-27
> 目标: 解决直播模块对星阵围棋(19x19.com)的单一依赖，从源头获取直播数据

---

## 一、核心发现：围棋直播数据链的真相

### 完整数据链

```
比赛现场
  ├─ 电子棋盘 (IZIS等) ──→ 电子棋盘云平台 ──→ OGS/gogameslive.eu
  └─ 人工录入 ──→ 主办方内部系统
                      ↓
              ┌───────────────────┐
              │ 主办方/围棋协会     │
              │ (中国/日本/韩国)    │
              └───────┬───────────┘
                      ↓
              ┌───────────────────┐
              │ 授权播出平台        │  ← 第一层下游
              │ 野狐/弈客/幽玄/    │
              │ Baduk TV/新浪      │
              └───────┬───────────┘
                      ↓
              ┌───────────────────┐
              │ 二次传播平台        │  ← 第二层下游
              │ 星阵围棋(19x19)    │
              │ 各种APP/网站       │
              └───────────────────┘
```

### 关键事实：没有任何围棋协会提供公开的直播数据 API

| 围棋协会 | 是否提供公开数据API | 官方播出渠道 |
|---------|------------------|------------|
| 中国围棋协会 | **否** — 无公开API，cwql.org.cn有内部管理系统(需登录) | 委托给野狐围棋、弈客围棋等商业平台 |
| 日本棋院 | **否** — 通过幽玄の間播出，付费墙($20/月)，无公开API | 幽玄の間 (u-gen.nihonkiin.or.jp) |
| 韩国棋院 | **否** — 通过Baduk TV(视频)和WBaduk(私有协议)播出 | Baduk TV (tvbaduk.com), WBaduk |

**这意味着：不存在"直接从围棋协会获取数据"的技术路径。** 协会本身不提供数据服务，而是委托给授权平台。

## 二、各平台深度分析

### A. 弈客围棋 (YikeWeiqi) — 最有潜力的替代源 ✅ 已验证

**可编程接入方式（已验证可用）：**

1. **api-new 列表+详情 API（推荐，已验证）：**
   ```
   列表: GET https://api-new.yikeweiqi.com/v1/golives?page=1&page_size=50&status=2
   详情: GET https://api-new.yikeweiqi.com/v1/golives/{id}
   过滤: status=2(直播中), type=0(职业赛)
   ```
   需要HMAC-SHA1认证（密钥来自前端JS，等同于公开）：
   ```python
   AppKey = "3396jtzhK57XhJom"
   AppSecret = "hfdSXRKm0DQyLmNXmNCNkZpjy2o5q1Hk"
   CheckSum = SHA1(AppSecret + Nonce + CurTime)
   Headers: AppKey, CurTime, CheckSum, Nonce, usertoken=-1, version=96813, Platform=web, app_id=yike, organization=yike
   ```
   验证结果：返回34场直播对局，含完整SGF、棋手信息(姓名/国籍/头像)、手数、胜率等。

2. **Share API 单局查询（免认证备选）：**
   ```
   GET https://share.yikeweiqi.com/golive/detail?id={ID}
   Headers: Accept: application/json, X-Requested-With: XMLHttpRequest
   ```
   无需认证，返回完整SGF。无列表API，但ID连续（每天约50-90个新ID），可扫描发现。

3. **WebSocket(Centrifugo)：** `wss://golive-api.yikeweiqi.com/connection/websocket`
   ❌ 已验证：需要App认证token，**匿名连接不可用**。

**是否为一手源？** 不是。弈客从主办方获取数据，但它是中国围棋协会的授权平台之一，覆盖中国赛事和国际大赛。比星阵围棋更接近源头一层。

**法律风险评估：**
- 棋步数据本身不受著作权保护（最高检公报：棋谱是事实记录，不具独创性）
- api-new 的认证密钥来自公开的前端JS包，等同于公开接口
- Share API 完全无需认证
- 非商业使用风险极低
- ⚠️ 如商业化需谨慎：《体育法》第52条(2023年生效)禁止营利性未授权采集赛事信息

### B. Pandanet-IGS — 日本职业赛的可靠源 ✅ 已验证

**接入方式（已验证）：** TCP Telnet协议，端口7777
```
连接: igs.joyjoy.net:7777
Guest登录(无需密码) → toggle client true → games (列出对局) → observe {game_id} (观战)
```

**Client模式棋步格式（机器可解析）：**
```
15 Game 227 I: j987654321 (12 450 9) vs cau (15 359 5)
15 249(W): J6 H6        ← 白249手下在J6，提子H6
15 250(B): E15           ← 黑250手下在E15
```
坐标格式: 字母A-T(跳过I) + 数字1-19

**验证结果：**
- Guest登录成功，分配临时账号 (guest3344)
- `games` 命令返回~40场活跃对局
- `observe` 命令可实时接收棋步推送
- `moves` 命令可获取完整对局记录
- ⚠️ 当前无职业赛转播——都是业余对局(最高7d)。职业赛转播是定期赛事(如Pandanet杯)

**覆盖赛事：** 日本主要头衔战（名人战、本因坊战、棋圣战、十段战等），但非持续性转播

**是否为一手源？** Pandanet ≠ 幽玄の間，但确实直接从日本棋院获得授权转播日本职业赛。比从星阵围棋获取更接近源头。

**法律风险：** Guest观战是平台提供的功能，非破解行为。非商业使用风险极低。

**现有实现参考：**
- qGo (C++): 最完整的IGS协议实现，含完整消息码表
- pandapy (Python): 简单的Python IGS客户端
- 无现代Python asyncio库，需自建

### C. OGS (online-go.com) — 最开放但覆盖有限

**接入方式：** REST API + WebSocket
```
WebSocket: wss://online-go.com/ws
观战: ["game/connect", {"game_id": 123456}]
接收: game/<id>/move (实时推送)
```

**职业赛覆盖？** 有限。OGS职业赛来源于：
1. 志愿者手动中继（Demo Board）
2. IZIS电子棋盘自动传输（欧洲赛事，通过gogameslive.eu）

**是否实时？** 是。WebSocket推送，延迟亚秒级。

**法律风险：** 最低。完全公开的文档化API，匿名观战。

### D. 野狐围棋 (Fox Weiqi) — 覆盖最全但难接入

**非官方API (OpenFoxWQ):**
```
Base: https://foxwq-8e6797d8dbb9.herokuapp.com/api/v1
Headers: X-APP-ID, X-API-KEY (需联系维护者获取)
GET /games/{id} → 含SGF的JSON
GET /top_games → 顶级对局列表
```

**覆盖：** 几乎所有职业赛事。中国围棋协会授权的主要平台。

**法律风险：** 高。API是逆向工程产物，Tencent未官方授权。Heroku代理不稳定。

### E. 新浪围棋 (Sina Go) — 衰退但仍有价值

**接入方式：** 直接HTTP下载SGF
```
URL模式: http://sinago.com/cgibo/YYYYMM/filename.sgf
棋谱列表: http://sinago.com/qipu/new_gibo.asp
```

**状态：** 正在衰退，但历史数据仍可访问。SSL证书有问题。

### F. 视频直播 + AI棋盘识别 — 最接近真正源头

**可用视频源：**
| 来源 | 覆盖 | 访问方式 |
|------|------|---------|
| 天元围棋频道 (weiqi.com.cn) | 中国~150场/年 | Bilibili/优酷直播 |
| Baduk TV (tvbaduk.com) | 韩国职业赛 | YouTube直播 |
| Bilibili围棋区 | 混合 | 直播/录播 |

**AI棋盘识别技术现状：**
| 方案 | 准确率 | 速度 | 可用性 |
|------|-------|------|-------|
| 2025台湾论文 (DETR+ViT) | F1=0.988, 全局82.56% | 377.5 FPS | 论文级，代码未确认公开 |
| Kifu Snap (商业App) | 良好(条件好时) | 实时 | 非开源，但可用 |

**这条路径的意义：** 电视直播是比赛的**官方授权播出**，从视频中提取棋步相当于"看电视记录棋谱"——这是每个围棋爱好者都在做的事情，不存在法律障碍。

## 三、法律风险总结

### 棋步数据的法律性质

1. **棋步本身不受著作权保护** — 最高人民检察院公报分析：棋谱是事实记录，不具独创性
2. **加注解说的棋谱受保护** — 评论、分析文字属于创作
3. **视频直播受保护** — 但从视频中提取棋步（如同手记棋谱）不侵犯著作权
4. **中国《体育法》第52条(2023)** — 禁止营利性未授权采集赛事信息。非营利使用不受限

### 各方案法律风险排序（从低到高）

| 方案 | 风险 | 原因 |
|------|------|------|
| 从公开视频提取棋步 | ★☆☆☆☆ | 等同于人工看电视记棋谱 |
| OGS公开API | ★☆☆☆☆ | 官方文档化API，匿名访问 |
| Pandanet-IGS Guest观战 | ★★☆☆☆ | 平台提供的Guest功能 |
| 弈客围棋 Share API | ★★☆☆☆ | 公开可访问，非商业OK |
| 新浪围棋 SGF下载 | ★★☆☆☆ | 公开可下载的文件 |
| 19x19.com API | ★★★☆☆ | 逆向工程的非公开API |
| Fox Go 非官方API | ★★★★☆ | 逆向工程 + 第三方Heroku代理 |

## 四、推荐方案：分层多源架构

### 设计原则

1. **合法性优先** — 优先使用公开/授权的数据源
2. **接近源头** — 尽可能从授权平台而非二次传播平台获取
3. **多源冗余** — 任何单一源失效不影响整体
4. **按赛事路由** — 不同赛事使用最佳源

### 架构图

```
┌─────────────────── 数据源层 ──────────────────────┐
│                                                     │
│  中国赛事:    弈客api-new(主) ←→ 19x19(备)          │
│  日本赛事:    Pandanet-IGS(telnet) ←→ 弈客 ←→ 19x19│
│  韩国/国际:   弈客api-new(主) ←→ 19x19(备)          │
│  欧洲赛事:    OGS WebSocket (IZIS电子棋盘)          │
│  (未来)视频:  AI棋盘识别 ← Bilibili/YouTube         │
│                                                     │
│  ←→ 表示互为 fallback                               │
└─────────────────────┬───────────────────────────────┘
                      ↓
          ┌───────────────────────┐
          │    SourceManager      │
          │  - 源适配器注册        │
          │  - 赛事→源路由        │
          │  - 去重(棋手+日期)     │
          │  - 健康检查/自动切换   │
          └───────────┬───────────┘
                      ↓
          ┌───────────────────────┐
          │  现有 LiveService     │  ← 最小改动
          │  (缓存/分析/API)      │
          └───────────────────────┘
```

### 实施路线

#### Phase 1: 弈客围棋接入 + 源管理框架 (2-3周)

**为什么先弈客？（已验证可行）**
- api-new 提供完整列表+详情API（HMAC认证，密钥已知）
- 返回完整SGF + 棋手信息 + AI胜率，数据质量高
- 覆盖中国赛事+国际大赛（与19x19.com重叠最多）
- 可立即形成双源架构

**具体工作：**
1. 实现 `YikeWeiQiSource` 适配器
   - 列表: `GET /v1/golives?status=2&type=0` (HMAC认证)
   - 详情: `GET /v1/golives/{id}` → SGF
   - 备选: `share.yikeweiqi.com/golive/detail?id={ID}` (免认证)
2. 实现 `SourceManager` (源注册、路由、去重、健康检查)
3. 修改现有 `LivePoller` 支持多源
4. 保留 `XingZhenSource` 作为 fallback

#### Phase 2: Pandanet-IGS接入 (2周)

**为什么？**
- 日本职业赛的授权转播平台
- Telnet协议成熟，有完整文档
- Guest可观战，无需付费
- 扩展覆盖到日本头衔战

**具体工作：**
1. 实现 asyncio 版 IGS 协议客户端
2. 实现 `PandanetSource` 适配器
3. IGS坐标 → SGF 转换
4. 赛事路由：日本赛事优先走 Pandanet

#### Phase 3: 视频AI识别（探索性）(4-6周)

**为什么？**
- 最接近真正源头（官方电视直播）
- 法律风险最低
- 技术上最有创新性
- 可覆盖所有有视频直播的赛事

**具体工作：**
1. 调研并测试 Kifu Snap 的 OGS广播功能
2. 评估2025台湾论文的方法是否可复现
3. 如果可行，构建轻量级棋盘识别管道
4. 对接 Bilibili/YouTube 直播流

#### Phase 4: 扩展覆盖 (持续)
- OGS WebSocket 集成（欧洲赛事 via IZIS）
- 韩国赛事源探索
- 社区贡献机制

## 七、最终决策 (2026-03-27)

### 确定的实施路线

| Phase | 数据源 | 覆盖 | 状态 |
|-------|-------|------|------|
| **Phase 1** | 弈客围棋 api-new (主) + 19x19.com (备) | 中国赛事 + 国际大赛 | ✅ 已验证，待实施 |
| **Phase 2** | Pandanet-IGS Telnet | 日本主要头衔战 | ✅ 已验证，待实施 |
| **Phase 3** | AI视频棋盘识别 (探索性) | 所有有视频直播的赛事 | 待调研 |
| **后续** | 幽玄の間 (如需完整日本覆盖) | 全部日本赛事 | 视需要评估 |

### 架构决策

- **SourceManager 多源框架**: 源适配器注册 + 赛事路由 + 去重 + 健康检查 + 自动fallback
- **弈客围棋**: 替代19x19.com成为中国/国际赛事的主源
- **Pandanet-IGS**: 日本赛事源，免费Guest Telnet
- **19x19.com**: 降级为fallback，不再作为主源
- **幽玄の間赛程表**: 可作为日本赛事的赛程发现数据源(公开抓取)
- **幽玄の間赛后SGF**: 可作为赛后棋谱归档的补充源(公开下载)
